# Contributing

Thanks for contributing.

## Local setup

```bash
composer update
composer test
composer stan
composer lint
```

## Contribution rules

- Keep package framework-agnostic.
- Do not add heavy runtime dependencies.
- Add/adjust tests with behavior changes.
- Update README/ARCHITECTURE/docs when public behavior changes.

## Commit style

Use conventional commit prefixes:

- `feat:`
- `fix:`
- `docs:`
- `test:`
- `refactor:`
- `chore:`
